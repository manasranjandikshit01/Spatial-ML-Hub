1. Overall goal

Build a complete, well‑structured repo that does the following:

Objective‑1: Surface AQI estimation

Create a data pipeline that takes as input:

Columnar satellite pollutants (Sentinel‑5P/TROPOMI: NO2, SO2, CO, O3, HCHO).

Aerosol Optical Depth from INSAT‑3D.

Meteorological reanalysis (ERA5 or IMDAA or MERRA‑2: temperature, humidity, wind, etc.).

Ground CPCB monitoring data (PM2.5, PM10, NO2, O3, SO2, CO).

Build a training dataset that aligns all of these on a daily time scale and a common spatial grid over India.

Train a deep learning model (start with a strong baseline and also provide a CNN‑LSTM version) that predicts surface PM2.5 (and optionally other pollutants) from the satellite + met features.

Compute Indian AQI from the predicted pollutants and from observed CPCB data.

Evaluate with RMSE, MAE, R, R² and generate daily gridded AQI maps over India.

Objective‑2: HCHO hotspots during biomass burning

Create a pipeline for Sentinel‑5P TROPOMI HCHO data over India.

Integrate MODIS/VIIRS fire counts (FIRMS) and the same reanalysis wind fields.

Identify biomass burning seasons (forest fires and crop residue burning) and detect HCHO hotspots using percentile thresholds and clustering.

Analyse correlations between HCHO and fire counts; show how wind transports HCHO from fire regions to downwind areas.

Produce maps and time series visualizations for Indo‑Gangetic Plain, forest fire zones, etc.

Package everything into a repo with:

Clean, modular Python code.

Clear configuration files.

A minimal web UI or dashboard (e.g., Streamlit) to showcase AQI and HCHO hotspot maps.

Step‑by‑step documentation so a teammate can run it end‑to‑end.
